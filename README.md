# industry-depth-research-skill
A structured industry research skill distilled from How to Quickly Understand an Industry, designed to generate decision-ready industry analysis reports.

#行业深度研究分析 Skill

#一个基于《如何快速了解一个行业》方法论提炼而成的行业研究分析 Skill。

它的目标不是复述原书内容，而是把书中的行业研究框架转化成一套可执行、可复用、可输出报告的分析流程，帮助用户在面对一个陌生行业、市场、赛道或投资机会时，快速建立结构化认知，并生成更接近“决策可用”的行业研究报告。

项目简介

industry-depth-research 是一个面向行业研究场景的 Skill。

当用户提出类似以下问题时，可以触发该 Skill：

快速了解一个行业
深度分析某个行业
写一份行业研究报告
判断某个赛道是否值得进入
分析某个市场的规模、商业模式和竞争格局
评估某个行业的投资机会、创业机会或市场进入机会

该 Skill 会围绕行业边界、商业模式、生命周期、可行性、规模性、防守性、盈利性、估值、外部因素和景气度等模块展开分析，并将结论组织成一份结构化行业报告。

方法来源

本项目的方法框架主要来自我对《如何快速了解一个行业》一书的阅读、理解和实践提炼。

我将书中关于行业研究的核心思路整理成一套可执行的 Skill，包括：

如何定义研究对象和行业颗粒度
如何使用商业模式画布理解行业运行机制
如何判断产业生命周期
如何从可行性、规模性、防守性、盈利性等角度拆解行业
如何结合估值、外部因素和景气度进行动态判断
如何用假设驱动、金字塔原理和结构化表达完成研究输出

本项目不包含原书正文内容，也不用于替代原书阅读。它更像是一个基于个人理解整理出来的行业研究执行工具。

核心能力
1. 定义研究对象和颗粒度

在开始分析之前，Skill 会先明确行业边界，包括：

研究的行业或赛道是什么
分析的地域范围是什么
时间周期是什么
包含哪些产品、服务或细分市场
不包含哪些相邻市场或替代品
市场规模和竞争格局可能存在哪些口径风险

这一步的作用是避免在行业研究中出现“概念很大、结论很虚”的问题。

2. 构建商业模式画布

Skill 会优先使用商业模式画布，分析行业如何创造、传递和获取价值，包括：

客户细分
价值主张
渠道
客户关系
收入来源
核心资源
关键活动
关键合作方
成本结构

商业模式画布是后续判断可行性、规模性、防守性和盈利性的基础。

3. 判断产业生命周期

Skill 会根据行业渗透率、增长速度、商业模式成熟度和竞争状态，判断行业处于：

导入期
成长期
成熟期
衰退期

不同生命周期阶段会影响分析重点：

导入期更重视可行性
成长期更重视规模性
成熟期更重视防守性和盈利性
衰退期更重视替代风险和估值陷阱
4. 行业研究分析框架：可行性

用于判断一个行业或商业模式是否真的成立。

重点分析：

需求是否真实存在
用户是否愿意付费
销售是否可行
单位经济模型是否成立
成本是否可以标准化或规模化
是否存在关键反证
5. 行业研究分析框架：规模性

用于判断行业的市场空间和增长潜力。

重点分析：

TAM：总市场空间
SAM：可服务市场
SOM：可获得市场
市场规模测算口径
需求导向、供给导向或供需匹配测算方法
乐观、中性、悲观情景假设
6. 行业研究分析框架：防守性

用于判断行业或企业是否具备护城河，以及护城河是否可持续。

重点分析：

资源垄断
技术壁垒
数据壁垒
网络效应
渠道控制
供应链控制
替代品威胁
护城河是变宽、变窄，还是会被绕开
7. 行业研究分析框架：盈利性

用于判断行业中到底是谁赚钱，以及利润如何分配。

重点分析：

横向竞争格局
市场集中度
进入壁垒
价格战风险
产业链上下游议价权
各环节利润率
产能周期、库存周期和供需关系
8. 行业研究分析框架：估值

当用户的问题涉及投资、融资、并购或二级市场定价时，Skill 会进一步分析估值问题。

重点分析：

好行业是否等于好价格
投资者是否有能力买
投资者是否愿意买
资产的赔率和胜率
交易是否拥挤
估值与基本面之间是否匹配
9. 行业研究分析框架：外部因素

Skill 会使用 PEST 框架分析行业外部变量：

Political：政策因素
Economic：经济因素
Socio-cultural：社会文化因素
Technological：技术因素

重点不是罗列外部因素，而是说明这些因素如何影响行业基本面、竞争格局和估值。

10. 行业研究分析框架：景气度

Skill 会为行业建立一组可跟踪的高频指标，用于持续验证判断。

例如：

上游价格
下游需求
开工率
库存水平
订单量
渗透率
政策变化
用户行为变化
龙头公司经营数据

每个指标都会说明其业务逻辑、更新频率、数据来源建议，以及什么变化会改变原有结论。

输出特点

本 Skill 的输出强调：

先给核心判断，而不是堆砌资料
先定义行业边界，再进行市场规模和竞争分析
区分事实、假设和判断
明确数据口径、证据质量和不确定性
避免单一来源带来的过度确定性
避免模板化分析，强调结论对决策的意义
最后给出可持续跟踪的指标看板
适用场景

该 Skill 适用于：

行业研究
市场调研
投资分析
创业方向判断
产品战略分析
商业计划书准备
咨询项目分析
PE/VC 尽调初筛
赛道机会拆解
职业学习和行业入门
使用示例

你可以这样提问：

请帮我深度分析中国预制菜行业，并生成一份行业研究报告。

也可以这样提问：

我想判断 AI 教育这个赛道值不值得创业，请用行业研究分析框架帮我拆解。

或者：

请分析新能源汽车充电桩行业的市场规模、竞争格局、盈利性和景气度指标。

Skill 会根据用户目的，输出相应的行业分析结果。

设计原则
1. 不做泛泛而谈的行业介绍

Skill 不追求“看起来完整”，而是追求“对决策有帮助”。

每一个模块都需要回答：

这个结论对用户判断行业、进入市场、投资或制定策略有什么影响？

2. 不把框架当成填空题

框架只是分析工具，不是结论本身。

Skill 会避免机械填模板，而是尽量解释每个判断背后的业务逻辑、证据链和不确定性。

3. 不把市场规模当成万能答案

市场大不等于机会好。

Skill 会在市场规模之外，同时分析商业模式、竞争结构、盈利能力、护城河、估值和景气度。

4. 不把短期增长误认为长期趋势

Skill 会通过生命周期、渗透率、供需关系和高频指标，判断行业增长是阶段性波动，还是结构性机会。

5. 不把 AI 输出当成事实

Skill 强调交叉验证和证据质量。

对于关键数据、市场规模、政策变化和公司经营情况，需要尽量多源验证；无法验证时，应明确标注为假设或待确认信息。
